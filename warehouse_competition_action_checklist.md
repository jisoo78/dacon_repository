## 1) 창고가 어떤 상황인지 (운영 흐름 해석)
아래 항목을 팀 회의에서 "현재 창고는 이런 메커니즘으로 지연이 발생한다"는 스토리로 설명한다.

- [ ] 주문 유입/난이도 변화
  - `order_inflow_15m`, `unique_sku_15m`, `avg_items_per_order`, `urgent_order_ratio`
  - 주문량이 늘고 SKU가 분산될수록 피킹 동선/처리 복잡도가 증가
- [ ] 로봇 가동/휴지/충전 균형
  - `robot_active`, `robot_idle`, `robot_charging`, `robot_utilization`
  - 가동률이 높아도 충전 대기나 재할당이 늘면 실제 처리량이 꺾일 수 있음
- [ ] 배터리/충전 병목
  - `battery_mean`, `low_battery_ratio`, `charge_queue_length`, `avg_charge_wait`
  - 저전력 로봇 증가 + 충전 대기열 증가가 지연의 선행 신호
- [ ] 통로 혼잡/충돌 리스크
  - `congestion_score`, `max_zone_density`, `blocked_path_15m`, `near_collision_15m`
  - 병목 구간 밀도 증가 시 이동 효율 저하 및 대기 증가
- [ ] 패킹/도크 처리 병목
  - `pack_utilization`, `loading_dock_util`, `outbound_truck_wait_min`, `dock_to_stock_hours`
  - 출고 말단 공정이 막히면 전체 리드타임이 연쇄적으로 지연
- [ ] 시스템/네트워크 지연
  - `wms_response_time_ms`, `scanner_error_rate`, `wifi_signal_db`, `network_latency_ms`
  - WMS/스캐너/네트워크 품질 저하가 작업 멈춤 또는 재시도 증가 유발
- [ ] 환경/시설 영향
  - `warehouse_temp_avg`, `humidity_pct`, `floor_vibration_idx`, `maintenance_schedule_score`
  - 환경과 설비 노후도는 안정성/오류율에 간접 영향
